How To Guide

Everything you need to install, configure, and get the most out of the Scalyn QA Assistant plugin for WordPress.

📖 ~15 min read 🏷 v1.4.7 📅 Updated June 2026

📥 Installation

Scalyn QA Assistant can be installed like any WordPress plugin. No technical knowledge required.

Download the plugin

Download the .zip file from Google Drive.

Upload in WordPress

Go to Plugins → Add New → Upload Plugin, select the .zip file, and click Install Now.

Activate the plugin

Click Activate and you'll be redirected to the setup wizard.

💡
Auto-Updates The plugin includes update detection. Go to Settings → Advanced to configure update notifications directly in WordPress.

✨ Setup Wizard

On first activation, the setup wizard guides you through initial configuration. You can also access it later from Settings → Wizard tab.

Select post types to scan

Choose which post types (Posts, Pages, Custom Post Types) should be included in QA audits.

Set quality thresholds

Define what scores count as "green" (good), "yellow" (needs work), and "red" (critical). Defaults: green ≥ 80, yellow ≥ 50.

Connect an AI provider (optional)

Enter your API key for OpenAI, Gemini, Claude, or another provider to enable AI metadata generation.

Run your first scan

The wizard offers to batch-scan all published content so your dashboard is immediately populated.

🔍 Page Audits

Page audits are the core feature of Scalyn QA. Each audit runs all six analyzers against a page and produces a detailed checklist of pass/warning/fail items.

Audit List View

Navigate to Scalyn QA → Page Audits to see all audited pages in a filterable table. Each row shows the page title, URL, overall score, and category breakdowns.

🎯 Scoring System

Every check is assigned a severity level that determines its weight in the overall score:

SeverityWeightExample
Critical3xBroken links, missing H1, 404 errors
Warning2xShort meta description, missing lazy loading
Info1xValid mailto links, working external links

Category Weights

The overall score is a weighted average of three categories:

CategoryWeightCovers
SEO40%Meta tags, schema, canonicals, keyword density
Content35%Word count, readability, heading structure, punctuation
Functionality25%Links, images, forms, buttons, accessibility

📝 Content Analyzer

Ensures your content meets quality standards for readability and structure.

Checks Performed

Word Count

Validates minimum word count (default: 300 words, configurable in Settings).

Heading Capitalization

Warns if headings start with lowercase letters instead of title case.

Paragraph Punctuation

Ensures paragraphs end with proper punctuation (. ! ? : ; etc.).

Short Paragraphs

Flags 4+ consecutive single-sentence paragraphs that may indicate poor formatting.

Readability Score

Integrates with Yoast/Rank Math scores, or falls back to Flesch Reading Ease (target: 60+).

Smart Features

🛡 SSRF Protection

Blocks private IP ranges and prevents DNS rebinding attacks during link checking.

🤖 Bot Awareness

Skips validation for sites known to block bots (Facebook, LinkedIn, Instagram).

💾 24h Caching

Results cached via WordPress transients to avoid re-checking identical URLs.

🔢 Rate Limited

Max 50 links per scan to prevent server overload on link-heavy pages.

🖼 Image Optimization

Catches image issues that hurt Core Web Vitals and user experience.

Broken Images

Validates src attributes and resolves local paths to the filesystem to confirm images exist.

Missing Dimensions

Warns if width and height attributes are missing — these are critical for preventing Cumulative Layout Shift (CLS).

Lazy Loading

Requires loading="lazy" on all images except the first (which should load eagerly for Largest Contentful Paint).

File Size

Flags images exceeding your configured limit (default: 900 KB). Configurable in Settings → Page Audits.

🔤 Heading Structure

Proper heading hierarchy is crucial for both SEO and accessibility.

H1 Validation

Ensures exactly one H1 per page. Detects both content H1s and theme-rendered title H1s via frontend request.

Hierarchy Check

Warns on skipped levels (e.g., H1 → H3 without H2). Headings should follow a logical order.

Empty Headings

Flags empty heading tags as critical issues — they confuse screen readers and hurt SEO.

Elementor Support

Full support for headings inside Elementor page builder content.

📋 Forms & Buttons

Catches accessibility and usability issues in forms and interactive elements.

Empty Buttons

Validates all buttons have visible text or aria-label attributes (WCAG compliance).

Placeholder Links

Warns about non-functional href="#" or javascript:void(0) links.

Form Submit Buttons

Validates every <form> element contains a submit button.

Popup Triggers

Validates popup trigger elements reference valid popup targets, including Elementor popups.

Configuration Steps

Select your provider

Choose your primary AI provider from the dropdown menu.

Enter your API key

Paste your API key. It will be stored with AES-256-CBC encryption — never in plain text.

Test the connection

Click Test Connection for instant validation feedback.

Set a daily rate limit

Configure the maximum AI requests per day to control API costs.

🔄
Automatic Fallback Configure a secondary provider. If your primary provider fails (rate limit, downtime), the system automatically retries with the fallback provider.

Using AI Meta Generation

Once configured, go to any Page Audit → Single View and click the Generate Meta button. You can:

  • Generate title only, description only, or both
  • Preview the AI output before applying
  • Apply directly to your active SEO plugin (Yoast, Rank Math, AIOSEO, or SEOPress)
  • View generation logs with token usage and response times

📄 Report Generation

Generate branded, client-ready QA reports that you can print to PDF.

How to Generate

Configure report settings

Go to Settings → Generate Report. Upload your company logo, toggle sections, and set the max pages to include.

Click Generate Report

A new browser tab opens with a self-contained HTML report.

Print to PDF

Use your browser's print dialog (Ctrl+P / Cmd+P) to save as PDF. The report is print-optimized.

Report Sections

  • Site metadata (name, URL, WP version, PHP version, generation timestamp)
  • Project score summary with visual indicators
  • Top 15 failing issues across the entire site
  • Launch checklist results with per-category scores
  • Per-page scores (up to 500 pages)
  • Custom company logo branding

❓ FAQ

Does the plugin slow down my site? ▼
No. Scalyn QA only runs in the WordPress admin area. Scans are triggered manually or on post save — nothing runs on the frontend for visitors. The frontend inspector only loads for logged-in admins.
Do I need an AI API key? ▼
No. AI metadata generation is completely optional. All 6 analyzers, the launch checklist, reports, and the frontend inspector work without any AI provider configured.
Can I use this on client sites? ▼
Absolutely. Scalyn QA was built specifically for agencies managing multiple WordPress sites. The report generator produces branded, client-ready PDF reports with your company logo.
What SEO plugins are supported? ▼
Yoast SEO, Rank Math, All in One SEO (AIOSEO), and SEOPress. The plugin auto-detects which one is active and integrates accordingly for metadata reading and writing.
Does it work with Elementor? ▼
Yes. Scalyn QA has full Elementor support. It extracts content via Elementor's builder API, validates headings and images within Elementor content, and even checks Elementor popup triggers.
How do I uninstall cleanly? ▼
Go to Settings → Advanced and enable "Delete all plugin data on uninstall." Then deactivate and delete the plugin — all options, post meta, and transients will be removed.
